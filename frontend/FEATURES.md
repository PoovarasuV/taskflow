# Features Documentation

## 1. Todo Management

Users can:

* Create Todos
* Update Todo Status
* Delete Todos
* View Todo Information

---

## 2. Search Functionality

Users can search tasks dynamically.

Features:

* Real-time filtering
* Search suggestions
* Click suggestion to open details page

---

## 3. Priority System

Every Todo supports:

* Low
* Medium
* High

Priority is visually represented using neon badges.

---

## 4. Progress Tracking

Application calculates:

* Total Tasks
* Active Tasks
* Completed Tasks
* Completion Percentage

---

## 5. Todo Details Page

Each Todo contains:

* Title
* Description
* Status
* Priority
* Creation Date
* Last Updated Date

Access Example:

```plaintext
/todo?id=<todo_id>
```

---

## 6. User Experience

Implemented:

* Multi-page navigation
* Cyberpunk interface
* Interactive animations
* Responsive layout

---

## 7. Data Persistence

Todos are stored inside:

```plaintext
backend/data/todos.json
```

Data remains available after server restart.
